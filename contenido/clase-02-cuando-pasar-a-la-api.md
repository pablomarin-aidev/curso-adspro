# Clase 2 — Cuándo un negocio ya debería pasar a la API

**Módulo 2 · FUNDAMENTOS · ~7 min**

### No es cuestión de tamaño, es cuestión de dolor

Muchos negocios creen que la API de WhatsApp es "solo para empresas grandes". La realidad es distinta: lo que define el salto no es tu facturación ni cuántos empleados tienes, sino un conjunto de señales operativas muy concretas. En esta clase aprendes a reconocerlas y a medir, con tus propios números, si tu negocio ya cruzó la línea.

## Lo que aprenderás
1. Las 6 señales que indican que tu negocio ya necesita la API de WhatsApp (y por qué el tamaño no es una de ellas).
2. Cómo hacer un cálculo simple de ROI para saber cuándo el costo de NO tener API supera el de tenerla.
3. Cómo aplicarte un autodiagnóstico tipo scorecard para obtener una respuesta objetiva: "¿ya necesito API o todavía no?".

## Las 6 señales de que ya necesitas la API

No necesitas cumplirlas todas. Con identificarte en 2 o 3, ya es momento de considerar el salto.

| # | Señal | Síntoma que ves en el día a día | Por qué el WhatsApp normal se queda corto |
|---|-------|--------------------------------|-------------------------------------------|
| 1 | **Alto volumen de chats** | Entran más mensajes de los que alcanzas a responder; clientes preguntando "¿siguen ahí?". | No permite automatizar ni repartir la carga de forma seria. |
| 2 | **Varias asesoras atendiendo** | Dos o más personas peleándose un solo celular o pasándose el teléfono. | Un solo número no puede abrirse en varios dispositivos a la vez sin bloquearse. |
| 3 | **Necesidad de automatizar** | Copias y pegas los mismos mensajes decenas de veces al día. | No hay bienvenidas, menús ni respuestas automáticas nativas para ventas. |
| 4 | **Seguimiento de leads** | No sabes a cuántos clientes dejaste sin respuesta ni cuándo volver a contactarlos. | El seguimiento vive en la memoria de la asesora, no queda registrado. |
| 5 | **Campañas pagas que generan conversaciones** | Inviertes en pauta que manda gente a WhatsApp, pero no mides qué campaña vendió. | No conecta la conversación con la campaña ni con la venta. |
| 6 | **Desorden / pérdida de chats** | Chats sin responder enterrados, clientes duplicados, "nadie me contestó". | No hay bandeja unificada, historial ni asignación de conversaciones. |

### El detalle de cada señal

- **Volumen:** el problema no es tener muchos chats, es que se acumulen sin poder responderlos a tiempo. La demora en la primera respuesta es la fuga más cara.
- **Equipo:** cuando más de una persona atiende, el celular compartido se convierte en un cuello de botella. La API permite que cada asesora trabaje desde su propio equipo con un único número.
- **Automatización:** tareas repetitivas (saludos, menús, preguntas frecuentes, envío de catálogo) consumen horas que deberían ir a atender y cerrar ventas.
- **Seguimiento:** sin registro, cada lead que no se contactó de nuevo es una venta que se enfría. La API deja rastro de quién escribió, en qué quedó y cuándo retomar.
- **Campañas pagas:** cada chat que se cae es dinero de pauta desperdiciado. La API permite atribuir conversaciones y ventas a la campaña que las originó.
- **Desorden:** la señal más clara de todas es un cliente reclamando que nadie lo atendió, y que sea verdad.

## El cálculo simple de ROI

No necesitas una fórmula complicada. La lógica es esta:

1. **Estima tus chats por día.** ¿Cuántas conversaciones nuevas entran en una jornada normal? (cifra a insertar según la fuente del curso)
2. **Cuenta cuántas personas los atienden.** Divide los chats entre las asesoras disponibles.
3. **Compara con la capacidad real de atención.** Cuando el volumen de chats supera lo que tu equipo alcanza a atender bien, empiezas a perder ventas por demora y por chats caídos.

> **Regla mental:** cuando el costo de las ventas que pierdes por desorden y demora es mayor que el costo de la herramienta, la API deja de ser un gasto y pasa a ser una inversión que se paga sola. (cifra a insertar según la fuente del curso)

## Recurso de la clase: Scorecard "¿Mi negocio ya necesita API?"

Autodiagnóstico puntuable. Responde cada criterio con el puntaje que corresponda y suma el total.

| # | Criterio | 0 pts | 1 pt | 2 pts | Tu puntaje |
|---|----------|-------|------|-------|------------|
| 1 | Chats nuevos que entran por día | Pocos | Moderados | Muchos, no dan abasto | |
| 2 | Personas atendiendo el WhatsApp | 1 | 2 | 3 o más | |
| 3 | ¿Repites mensajes manualmente? | Casi nunca | A veces | Todo el día | |
| 4 | ¿Haces seguimiento ordenado a los leads? | Sí, con registro | A medias | No, se me pierden | |
| 5 | ¿Inviertes en pauta que llega a WhatsApp? | No | Poco | Sí, constante | |
| 6 | ¿Se te pierden o quedan chats sin responder? | Nunca | A veces | Con frecuencia | |
| 7 | ¿Un cliente ha reclamado que nadie lo atendió? | Nunca | Alguna vez | Sí, y era cierto | |
| | **Total** | | | | **/14** |

**Cómo leer tu puntaje:**

| Rango | Lectura | Recomendación |
|-------|---------|---------------|
| Bajo | Todavía manejas el WhatsApp normal sin dolor real. | Aún no necesitas la API; vuelve a medir en unos meses. |
| Medio | Ya sientes fricción en varios frentes. | Empieza a planear el salto a la API. |
| Alto | El dolor operativo ya te está costando ventas. | Necesitas la API ahora. |

## Para recordar
- La API no depende del tamaño de tu negocio, sino de señales de dolor operativo concretas: volumen, equipo, automatización, seguimiento, pauta y desorden.
- Con identificarte en 2 o 3 señales, ya es momento de considerar el salto.
- El scorecard te da una respuesta objetiva; llénalo con tus propios números antes de decidir.
