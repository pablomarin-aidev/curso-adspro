# Clase 8 — Tipos de plataformas (y los agentes de IA por dentro)

**Módulo 8 · PLATAFORMAS & AGENTES · ~6 min**
**Formato:** puros bullets, contenido completo de la clase (alternativa a `guiones/`).

- Muchos negocios eligen mal la herramienta de WhatsApp y después culpan a la IA de que "no vende".
- El problema casi nunca es la IA: compraron una cosa para un trabajo que esa cosa no hace.

## Lo que aprenderás
- Los seis tipos de plataforma —bandeja, automatización básica, builder, agente IA, todo-en-uno y proveedor técnico— para qué sirve cada uno y su techo.
- Cómo elegir por el trabajo que se necesita hacer hoy y el límite que se puede aguantar mañana.
- Qué hay por dentro de un agente de IA: el prompt, el handoff a humano, el tono humano y el debounce.

## Regla de oro
- No existe "la mejor plataforma", existe la mejor para lo que se necesita hacer hoy.
- Cada tipo resuelve un trabajo distinto y tiene un techo, un punto donde se queda corto.
- La clave no es comprar la más potente, sino la que hace el trabajo actual y aguanta hacia dónde se va.
- Cuando un límite empieza a estorbar, esa es la señal de subir de tipo.

## Los seis tipos de plataforma
- **1. Bandeja simple (inbox):** pantalla para leer y responder WhatsApp a mano, ordenados por chat. Centraliza la conversación. Límite: todo manual, sin humano no pasa nada.
- **2. Automatización básica (reglas):** bandeja con reglas "si pasa esto, haz esto" — respuestas por palabra clave, bienvenida, etiquetas. Límite: solo hace lo programado, si el cliente escribe distinto se queda corta.
- **3. Chatbot builder (flujos y botones):** conversaciones guiadas con menús y árboles de decisión. Límite: el cliente debe seguir el guion; si se sale, el bot se pierde y se siente robótico.
- **4. Agentes con IA:** entienden lenguaje natural, responden con contexto, consultan catálogo o CRM, pasan a humano cuando toca. Límite: hay que configurarlos bien; mal instruidos, inventan o prometen de más.
- **5. Todo-en-uno:** agente de IA + CRM + catálogo + campañas + seguimientos + métricas, todo conectado. Gestiona el ciclo completo del lead. Límite: se paga por cosas que quizá no se usen aún, curva de aprendizaje.
- **6. Proveedor técnico (API):** acceso a la API de WhatsApp para construir encima. Control total y personalización. Límite: se necesita equipo técnico; sin desarrollador, no hay nada usable.

## Los agentes de IA por dentro (4 piezas)
- **El prompt:** instrucciones del agente — quién es, qué vende, qué puede prometer y qué no, cómo responde. Aquí se gana o se pierde: buen prompt convierte, uno flojo inventa.
- **El handoff a humano:** regla de cuándo el agente suelta la conversación y avisa a una persona — cliente molesto, caso raro, o piden hablar con alguien. Mal puesto, quema clientes.
- **El tono humano:** que suene a persona, no a robot — frases cortas, natural, sin acartonarse. Se define en el prompt y cambia por completo la sensación del cliente.
- **Agrupar mensajes (debounce):** la gente escribe en pedacitos ("hola" / "quiero info" / "del azul"). El debounce espera un momento, junta los mensajes y responde una sola vez, como una persona.

## Recurso: tabla comparativa
- **Bandeja simple:** centraliza chats en equipo → límite: todo manual → para negocio chico, pocos chats.
- **Automatización básica:** respuestas por palabra clave/bienvenida/etiquetas → límite: no entiende, solo reacciona → para volumen de preguntas repetidas.
- **Chatbot builder:** flujos guiados con menús → límite: rígido y robótico si el cliente se sale del guion → para procesos estandarizables.
- **Agente con IA:** conversar, vender, calificar, pasar a humano → límite: mal hecho, inventa → para negocios que venden conversando.
- **Todo-en-uno:** ciclo completo (agente + CRM + campañas + métricas) → límite: se paga de más si no se usa todo → para negocio con volumen que escala ordenado.
- **Proveedor técnico:** control total vía API → límite: necesita equipo técnico → para empresas con área de tecnología o agencias.

## Para recordar
- No comprar por marca ni por precio: comprar por el trabajo que se necesita y el techo que se puede aguantar.
- Los seis tipos van de menos a más capacidad; cada uno tiene un límite que señala cuándo subir de tipo.
- Un agente de IA se resume en cuatro piezas: prompt, handoff, tono y debounce.
- El prompt es donde más se gana o se pierde: bueno vende, flojo inventa.
- El debounce hace que el agente espere y agrupe los mensajes, para responder como una persona.
