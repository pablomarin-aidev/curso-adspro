# Clase 4 — Cuándo un negocio ya debería pasar a la API

**Módulo 4 · FUNDAMENTOS · ~5 min**
**Formato:** puros bullets, contenido completo de la clase (alternativa a `guiones/`).

- La API de WhatsApp no es "solo para empresas grandes".
- Lo que define el salto no es la facturación ni el número de empleados, sino señales operativas concretas.

## Lo que aprenderás
- Las 6 señales que indican que un negocio ya necesita la API de WhatsApp (y por qué el tamaño no es una de ellas).
- Cómo hacer un cálculo simple de ROI: cuándo el costo de NO tener API supera el de tenerla.
- Cómo aplicar un autodiagnóstico tipo scorecard para responder: "¿ya necesito API o todavía no?".

## Las 6 señales de que ya se necesita la API
- No hay que cumplirlas todas — con 2 o 3 ya es momento de considerar el salto.
- **1. Alto volumen de chats**
  - Síntoma: entran más mensajes de los que se alcanzan a responder; clientes preguntando "¿siguen ahí?".
  - Por qué falla el WhatsApp normal: no permite automatizar ni repartir la carga en serio.
- **2. Varias asesoras atendiendo**
  - Síntoma: dos o más personas peleándose un solo celular o pasándose el teléfono.
  - Por qué falla: un solo número no puede abrirse en varios dispositivos a la vez sin bloquearse.
- **3. Necesidad de automatizar**
  - Síntoma: copiar y pegar los mismos mensajes decenas de veces al día.
  - Por qué falla: no hay bienvenidas, menús ni respuestas automáticas nativas para ventas.
- **4. Seguimiento de leads**
  - Síntoma: no saber a cuántos clientes se dejó sin respuesta ni cuándo volver a contactarlos.
  - Por qué falla: el seguimiento vive en la memoria de la asesora, no queda registrado.
- **5. Campañas pagas que generan conversaciones**
  - Síntoma: se invierte en pauta que manda gente a WhatsApp, pero no se mide qué campaña vendió.
  - Por qué falla: no conecta la conversación con la campaña ni con la venta.
- **6. Desorden / pérdida de chats**
  - Síntoma: chats sin responder enterrados, clientes duplicados, "nadie me contestó".
  - Por qué falla: no hay bandeja unificada, historial ni asignación de conversaciones.

## El detalle de cada señal
- **Volumen:** el problema no es tener muchos chats, es que se acumulen sin poder responderlos a tiempo. La demora en la primera respuesta es la fuga más cara.
- **Equipo:** cuando más de una persona atiende, el celular compartido es cuello de botella. La API permite que cada asesora trabaje desde su propio equipo con un único número.
- **Automatización:** tareas repetitivas (saludos, menús, FAQ, envío de catálogo) consumen horas que deberían ir a cerrar ventas.
- **Seguimiento:** sin registro, cada lead no recontactado es una venta que se enfría. La API deja rastro de quién escribió, en qué quedó y cuándo retomar.
- **Campañas pagas:** cada chat que se cae es dinero de pauta desperdiciado. La API permite atribuir conversaciones y ventas a la campaña de origen.
- **Desorden:** la señal más clara — un cliente reclamando que nadie lo atendió, y que sea verdad.

## Recurso: Scorecard "¿Mi negocio ya necesita API?"
- Autodiagnóstico puntuable (0, 1 o 2 puntos por criterio, total sobre 14):
  - Chats nuevos por día: pocos (0) / moderados (1) / muchos, no dan abasto (2).
  - Personas atendiendo el WhatsApp: 1 (0) / 2 (1) / 3 o más (2).
  - ¿Repites mensajes manualmente?: casi nunca (0) / a veces (1) / todo el día (2).
  - ¿Haces seguimiento ordenado a los leads?: sí, con registro (0) / a medias (1) / no, se me pierden (2).
  - ¿Inviertes en pauta que llega a WhatsApp?: no (0) / poco (1) / sí, constante (2).
  - ¿Se te pierden o quedan chats sin responder?: nunca (0) / a veces (1) / con frecuencia (2).
  - ¿Un cliente ha reclamado que nadie lo atendió?: nunca (0) / alguna vez (1) / sí, y era cierto (2).
- Cómo leer el puntaje:
  - Bajo → todavía se maneja el WhatsApp normal sin dolor real; aún no se necesita la API, volver a medir en unos meses.
  - Medio → ya hay fricción en varios frentes; empezar a planear el salto.
  - Alto → el dolor operativo ya está costando ventas; se necesita la API ahora.

## Para recordar
- La API no depende del tamaño del negocio, sino de señales de dolor operativo: volumen, equipo, automatización, seguimiento, pauta y desorden.
- Con identificarse en 2 o 3 señales, ya es momento de considerar el salto.
- El scorecard da una respuesta objetiva; llenarlo con los propios números antes de decidir.
